# 可访问性与输入

## 输入

- 键盘。
- 手柄。
- 可重绑定按键。
- 支持左右手方向键布局。
- 手柄左摇杆死区、灵敏度和移动速度曲线。

当前 SpellKard 已实现左右手键盘预设、单项按键轮换修改、按键校验、手柄左摇杆死区/灵敏度/曲线计算，并把模拟方向以 `analog_x/analog_y/analog_strength` 记录进本地输入状态，保证 replay JSON 不依赖引擎对象。

## 默认键位

- 移动：方向键/WASD。
- 射击：Z。
- Bomb：X。
- 低速：Shift。
- 卡牌：1-4。
- 暂停：Esc。

## 视觉可访问性

- 低闪烁模式。
- 简洁背景。
- 弹幕透明度调整。
- 判定点常显。
- 擦弹圈练习模式显示。
- 色弱友好弹幕配色预设。

## 音频可访问性

- 分组音量。
- 关键事件视觉提示。
- 可关闭高频擦弹音。

当前 SpellKard 已有 Master、Music、SFX、UI、Voice 分组设置，并在运行时创建/应用对应 Godot audio bus 音量。生产音效和主题音频仍待导入。

## 网络提示

- 显示 ping 和连接质量。
- 排位网络不佳时明确提示。
- 重连时不隐藏对局状态。

## 验收

玩家只使用键盘或手柄都能完整完成登录、匹配、对局、卡组和宝箱流程。Steam 商业版还应能完成创意工坊主题启用和库存查看。

短期验收补充：

- 主菜单只保留游玩、收藏、社区、玩家设置四个一级入口；考证/匹配从游玩进入，卡组/宝箱/Replay 从收藏进入，好友/推广/社交从社区进入。
- 玩家设置可进入输入、音频、画面子页。
- 手柄曲线、分辨率、窗口模式、FPS/VSync、分组音量、按键修改均可由 UI row 执行并通过 headless smoke 覆盖。
- UI row 需要携带稳定分区元数据，至少覆盖输入、手柄、按键、音频、音量、画面和可访问性，正式菜单场景复用这些分区组织设置页。
- UI row 需要携带稳定控件元数据，正式菜单场景应把按键/曲线/分辨率渲染为选择器，把音量/灵敏度/死区/震动/震屏渲染为滑条，把 VSync 和可访问性项渲染为开关。
- 设置类 UI row 需要携带可渲染值域：选择器提供候选项和当前索引，滑条提供当前值、最小值、最大值、步进和单位，开关提供布尔状态。
