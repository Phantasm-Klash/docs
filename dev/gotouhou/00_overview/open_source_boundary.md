# 开源与闭源边界

gotouhou 采用“开源核心 + 闭源 Steam 商业适配层”的结构。开源核心保证社区可以自建基础服务器、查看玩法规则并贡献客户端与基础服务端；Steam 商业层负责购买校验、平台库存、市场交易、创意工坊和平台风控，不进入 GitHub 开源仓库。

## GitHub 开源内容

开源仓库包含：

- Godot 客户端基础玩法代码。
- 基础 Nakama/Go 服务端。
- 账号注册和登录的非 Steam 实现。
- 开房大厅。
- 对战房间。
- 规则设置。
- 卡组保存。
- 基础掉落和本地/私服库存。
- Replay、练习、观战和基础排行榜。
- 原创主题文本和开源素材。
- i18n key、主题包接口和 Mod 加载接口。

## 闭源 Steam 商业层

闭源仓库包含：

- Steamworks SDK 接入。
- Steam 登录和 Session Ticket 校验。
- Steam App 购买/所有权验证。
- Steam 成就和统计。
- Steam 创意工坊上传、订阅、下载和审核策略。
- Steam Inventory Service 物品定义、掉落、开箱和交易属性。
- Steam 市场可交易/可售卖物品策略。
- 商业服风控、反刷、掉落频率和封禁联动。
- Steam Web API Key、publisher key、服务端密钥和运营后台敏感配置。

## 开源服务器版本

开源服务器版本只承诺基础联机能力：

- 自建账号注册。
- 开房大厅。
- 好友或房间码对战。
- 规则设置。
- 卡组保存。
- 基础库存。
- 非市场化掉落。

开源版本不包含：

- Steam 购买校验。
- Steam 市场交易。
- Steam Inventory 掉落。
- 商业服反刷策略。
- 官方赛季经济配置。
- 官方掉落概率和宝箱收益策略。

## 边界原则

- 玩法规则尽量开源，商业平台适配闭源。
- 开源版不能依赖 Steamworks SDK 才能运行。
- 闭源层通过接口适配开源核心，不修改核心玩法语义。
- 文档中涉及 Steam 的实现细节只写接口和边界，密钥、后台策略和风控参数不写入开源仓库。
