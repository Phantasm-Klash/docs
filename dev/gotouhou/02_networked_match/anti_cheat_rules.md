# 反作弊规则

## 客户端不可信

客户端只能提交：

- 输入方向。
- 低速、射击、Bomb 状态。
- 卡牌槽位请求。
- UI 操作请求。

客户端不能提交：

- 分数。
- 擦弹次数。
- 命中结果。
- 真实位置。
- 开箱结果。
- 奖励领取结果。

## 输入检查

- 输入 tick 必须单调。
- seq 不能重复或大幅跳跃。
- 同一 tick 不能提交互斥操作。
- card_slot 必须在手牌范围内。
- Bomb 必须有资源。

## 状态检查

- 卡组必须来自服务端保存快照。
- 卡牌必须在玩家库存中。
- 施放必须满足费用、冷却、目标和模式限制。
- 位置偏差超过阈值时以服务端为准并记录异常。

## 经济检查

- 开箱、奖励、分解、任务领取必须幂等。
- 每次经济变更记录来源。
- 异常收益进入审计表。

## 模式检查

- rank 分、评级资格、Boss 伤害、Boss 扣血、大逃杀排名和回合选卡结果必须由服务端计算。
- 考证模式：客户端不能声明通关、评级或 top 30% 资格。
- 大逃杀：客户端不能声明 3 选 1 候选卡，只能提交选择。
- 世界 Boss：客户端不能提交伤害结果、Boss 剩余血量或世界通告。
- 副本 Boss：客户端不能提交通关结果或星级结果。

## 处罚层级

- 单局异常：丢弃非法输入。
- 重复异常：取消对局收益。
- 严重异常：临时封禁排位。
- 确认作弊：账号封禁并保留审计记录。
