# 判定点与擦弹

## 判定层级

- 视觉半径：角色立绘或自机贴图大小，只负责表现。
- 受击判定半径：真实 hitbox，初版建议 2-4 逻辑像素。
- 擦弹半径：围绕判定点的 graze zone，初版建议 14-22 逻辑像素。
- 子弹判定半径：按弹种配置，不一定等于贴图半径。

## 命中规则

- 每个逻辑 tick 检测自机受击圆与子弹判定圆。
- 发生命中后进入无敌时间，并记录 hit 事件。
- Bomb 或无敌期间仍可擦弹与移动，但命中不重复结算。
- 网络对局中命中只以服务器结果为准。

## 擦弹规则

- 子弹进入擦弹圈且未命中时触发擦弹。
- 同一颗子弹默认只能给同一玩家计一次擦弹。
- 若需要长弹/激光持续擦弹，则使用每弹每玩家冷却 tick。
- 擦弹提供分数、符力和连段倍率。

## UI 表现

- 低速时显示判定点。
- 训练模式可显示擦弹圈和子弹判定圆。
- PVP 正式模式默认只显示判定点，避免画面过载。

## 调试数据

- 每局记录 graze_count、hit_count、near_miss_tick、max_combo。
- Replay 回放时允许显示擦弹触发点。
