# 01 Core STG Client

本阶段实现不依赖联机的本地 STG 核心。目标是先把手感、判定、擦弹、Bomb、弹幕、计分和 Replay 打磨稳定，再接入服务器权威模型。客户端 UI 文本必须使用 i18n key，不能在代码里硬编码“卡牌”“符卡”等主题词。

## 核心原则

- 客户端本地原型可以先自结算，但接口形状要方便迁移到服务端权威。
- 所有玩法逻辑使用固定 tick，渲染可以插值。
- Replay 从第一版就按确定性输入流设计。
- 性能问题必须用对象池、批量绘制和上限策略解决，不能依赖处理落。

## 最小可玩目标

- 自机可高速/低速移动。
- 低速显示判定点。
- 子弹可命中、可擦弹、可消除。
- 有基础射击、Bomb、得分和结算。
- 能保存并重放一局本地 Replay。
