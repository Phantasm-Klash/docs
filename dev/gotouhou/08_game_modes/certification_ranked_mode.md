# 考证评级模式

## 目标

考证模式用于给玩家授予阶段评级，并把评级作为匹配优先级。评级不是单纯排位分，而是玩家通过指定难度关卡后的资格证明。

## 核心流程

1. 玩家选择当前可参与的评级考证。
2. 服务端加载对应难度关卡和规则版本。
3. 玩家通过关卡后获得该阶段评级。
4. 匹配时优先匹配同一评级玩家。
5. 对战胜者增加该评级内 rank 分。
6. 当玩家 rank 分进入当前服务器该评级排行榜前 30%，可参与下一阶段评级考证。

## 评级数据

每个评级配置：

- rating_code。
- display_i18n_key。
- required_previous_rating。
- challenge_stage_id。
- pass_conditions。
- rank_score_floor。
- next_unlock_percentile，初版固定为 top 30%。

## 考证关卡

考证关卡是服务端权威挑战：

- 固定 seed 或每日 seed。
- 指定弹幕阶段。
- 指定可用卡池或禁卡。
- 指定通关条件，如存活、分数、命中上限、擦弹数。

## 匹配规则

- 优先同评级匹配。
- 等待超过阈值后可扩展到相邻评级。
- 若跨评级匹配，rank 分变化按评级差修正。
- 新评级玩家从该评级初始 rank 分开始。

## Rank 分

- 胜者增加 rank 分。
- 败者减少或少量减少，具体由规则配置。
- 断线、逃跑和异常对局按惩罚规则处理。
- 只统计同一服务器或同一赛季分区。

## 晋级资格

玩家满足以下条件可参加下一阶段评级：

- 已通过当前评级考证。
- 当前评级 rank 分位于服务器排行榜前 30%。
- 当前赛季没有封禁或异常状态。

## 结算

结算记录：

- rating_code。
- rank_score_delta。
- percentile_after。
- next_certification_unlocked。
- replay_id。

## 测试重点

- 同评级优先匹配。
- top 30% 资格计算。
- 排行榜更新幂等。
- 跨评级匹配分数修正。
- 断线惩罚不授予晋级资格。
