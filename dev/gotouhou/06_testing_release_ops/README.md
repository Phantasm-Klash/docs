# 06 Testing Release Ops

本阶段保证游戏可测试、可平衡、可上线、可排查。弹幕 PVP 的核心风险是延迟、公平性、Replay 非确定、卡牌失衡、经济漏洞、Steam 闭源层边界和主题包授权风险。

## 目标

- 本地开发环境一键启动。
- 自动化测试覆盖服务端核心逻辑。
- 多延迟场景可复现。
- Replay 可校验。
- 部署和监控有基础方案。
- 封测流程明确。
- 开源版和 Steam 商业版测试边界明确。

## 优先级

1. 对局确定性。
2. 延迟下的低速手感。
3. 经济幂等。
4. 卡牌平衡数据。
5. 线上监控和审计。
6. Steam 购买校验、Inventory、Workshop 和市场流程只在闭源测试中验证。
