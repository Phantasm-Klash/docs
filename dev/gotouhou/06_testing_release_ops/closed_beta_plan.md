# 封闭测试计划

## 阶段 A：本地单机

目标：

- 验证移动、低速、判定、擦弹、Bomb 和 Replay。

参与者：

- 开发者和少量内部测试。

## 阶段 B：局域网/开发服

目标：

- 验证 Nakama 连接、权威对局、输入同步和快照修正。

重点：

- 80ms 以下手感。
- 重连。
- 在线 Replay。

## 阶段 C：小规模内测

目标：

- 验证卡组、宝箱、奖励、排行榜和基础活动。
- 验证考证评级、大逃杀、世界 Boss、副本 Boss 的基础闭环。

规模：

- 50-200 人。

## 阶段 D：赛季 0

目标：

- 验证排位、公平性、活动节奏和运营工具。

规模：

- 500-2000 人，按服务器能力调整。

## 反馈收集

- 网络体验。
- 低速手感。
- 弹幕可读性。
- 卡牌公平性。
- 宝箱掉落压力。
- 主题包文本和素材替换。
- UI 易用性。
- 多模式规则理解成本。
- Boss 让渡卡牌体验。
- 大逃杀回合节奏。

## 上线门槛

- Replay 校验稳定。
- 经济无重复领取漏洞。
- 延迟测试达标。
- 素材授权完整。
- i18n 覆盖完整。
- Steam 商业版购买校验、Workshop、Inventory 和市场流程通过闭源验收。
- 监控和封禁流程可用。
